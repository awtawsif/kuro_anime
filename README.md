# Starlight Anime Hub — Terminal CLI

A terminal-based anime discovery and streaming tool. Search anime, browse currently airing shows, view details, download episodes, and stream them directly from your terminal.

## Features

- **Search** — Find any anime by title, get persistent short codes for instant lookups later
- **Currently Airing** — Browse the latest releases with pagination
- **Detailed Info** — Synopsis, genres, relations, recommendations, and metadata
- **Episode Listings** — Browse episodes with pagination and sort order
- **Streaming** — Interactive episode/quality picker → mpv
- **Download Links** — Extract playable video URLs for use with any player

## Quick Start

### Prerequisites
- Python 3.10+
- [mpv](https://mpv.io/) (required for `starlight watch`)

### Install

```sh
git clone <repo-url>
cd starlight-anime-hub
pip install -e .
```

### First commands

```sh
# Search for anime (generates short codes)
starlight search "One Piece"

# View details using the short code
starlight detail onpi

# Browse episodes and watch
starlight watch onpi

# See what's airing
starlight airing --page 1
```

## Identifier Formats

Every anime-taking command (`detail`, `episodes`, `watch`, `download`) accepts three interchangeable formats, resolved in order:

| Format | Example | Speed | API Call |
|--------|---------|-------|----------|
| **Short code** | `onpi` | Instant | None |
| **UUID** | `37aeb550-3ff8-a635-d1c6-e62729f426c1` | Moderate | Fetch title |
| **Slug** | `one-piece` | Slowest | Search + resolve |

**Short codes** are generated by `search` and `airing` commands. They persist in `~/.starlight/state.json` and are reused across sessions. Once generated, you never need to search for that anime again.

**Slugs** work without any prior state — just type the kebab-case title: `one-piece`, `frieren-beyond-journeys-end`, `sousou-no-frieren`. If the slug matches multiple results, you'll be prompted to pick one.

**UUIDs** are the raw session IDs from the web app, provided for backward compatibility.

## Commands

### `starlight search <query>`
Search anime by title. Displays results with short codes and saves the codes to state.

```sh
starlight search "Frieren"
# Code  Title                              Type  Episodes  Score  Status
# frbe  Frieren: Beyond Journey's End     TV    28        9.26   Finished Airing
# sono  Sousou no Frieren                 TV    28        8.63   Finished Airing
```

### `starlight airing [--page N]`
Browse currently airing anime, paginated. Assigns short codes for each entry.

```sh
starlight airing --page 1
# Code  Title           Episode  Fansub      Aired
# moea  Monster Eater   9        Amazon      2026-05-28
```

### `starlight detail <anime>`
Show full anime info: synopsis, metadata, relations, and recommendations.

```sh
starlight detail onpi       # by short code
starlight detail one-piece  # by slug
```

### `starlight episodes <anime> [--page N] [--sort episode_asc|episode_desc]`
List episodes with pagination.

```sh
starlight episodes onpi --page 1 --sort episode_desc
```

### `starlight watch <anime> [episode_id] [--episode / -e]`
Interactive episode and quality picker. Extracts the video URL from kwik.cx and streams it via mpv.

```sh
starlight watch onpi
# → shows episode list, prompt to choose (or enter range like 1-30)
# → shows qualities (1080p, 720p, etc.)
# → launches mpv
```

If `episode_id` is provided (the `session` field from `episodes`), skips the episode picker.

Use `--episode` / `-e` as an alternative to the positional argument:

```sh
starlight watch onpi --episode 778b7b85ae0f
```

### `starlight download <anime> [episode_id] [--episode / -e]`
Same flow as `watch` but prints the final video URL instead of launching mpv.

```sh
starlight download onpi
# Video URL: https://cdn.kwik.cx/file/.../video.mp4
```

## How Short Codes Work

Codes are generated from the first two alphabetic words in the title:

| Title | Code | Notes |
|-------|------|-------|
| "One Piece" | `onpi` | First 2 chars of "One" + "Piece" |
| "One Piece: The Movie" | `onpi2` | Collision → `onpi` → `onpi2` |
| "Monster Eater" | `moea` | First 2 chars of "Monster" + "Eater" |
| "Monogatari" | `mono` | Single word → first 4 chars |

Codes are saved to `~/.starlight/state.json` under the `codes` key. The reverse mapping (`sessions_to_codes`) ensures the same code is reused if the same anime appears in both search and airing results.

## State Persistence

All state lives in `~/.starlight/state.json`:

```json
{
  "codes": {
    "onpi": {
      "session_id": "37aeb550-...",
      "title": "One Piece"
    },
    "moea": {
      "session_id": "d83ba939-...",
      "title": "Monster Eater"
    }
  },
  "sessions_to_codes": {
    "37aeb550-...": "onpi",
    "d83ba939-...": "moea"
  },
  "kwik_cache": {}
}
```

Delete this file to reset all state.

## Architecture

```
starlight_cli/
  __init__.py    # Package marker
  cli.py         # CLI commands (click + rich), concurrent episode fetching, range input in picker
  api.py         # REST + scrape animepahe.pw (shared requests.Session, centralized _api_get wrapper)
  config.py      # HTTP headers and target URLs
  kwik.py        # kwik.cx JS dean-packer decryption → .m3u8 URL (lazy curl_cffi session, raw HTML fallback)
  player.py      # mpv subprocess wrapper
  state.py       # ~/.starlight/state.json read/write (in-memory cache, fcntl file locking, kwik cache 7-day TTL)
```

## Dependencies

Runtime:
- `requests` — HTTP client for animepahe.pw
- `curl_cffi` — TLS-fingerprinted HTTP client for kwik.cx
- `beautifulsoup4` + `lxml` — HTML parsing
- `click` — CLI framework
- `rich` — Terminal formatting

System:
- `mpv` — Video player (required for `starlight watch`)

## How Watching Works

```
starlight watch onpi
  → fetch_episode_list(onpi_session_id)
  → user picks episode
  → fetch_episode_streams(anime_id, episode_id)
  → user picks quality (1080p, 720p, etc.)
  → extract_hls_url(kwik_url)
     → check ~/.starlight/state.json cache (7-day TTL)
     → GET kwik.cx page (if not cached)
     → decrypt JS dean-packer obfuscation
     → extract .m3u8 URL
  → mpv --referrer=https://kwik.cx/ <video_url>
```

## Troubleshooting

**"No module named 'flask'"**: The CLI no longer depends on Flask. If you see this error, reinstall:

```sh
pip uninstall starlight-anime-hub
pip install -e .
```

**mpv not found**: `starlight watch` requires mpv. Install it:

```sh
# Debian/Ubuntu
sudo apt install mpv
# macOS
brew install mpv
# Windows
choco install mpv
```

**"Could not resolve"**: The identifier wasn't a short code, UUID, or matching slug. Try `starlight search <query>` first to generate short codes.
